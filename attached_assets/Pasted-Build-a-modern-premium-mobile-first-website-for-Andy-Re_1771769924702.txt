APPLICATION FORM (/apply)
Goal: multi-step form that matches site design (like the provided UI example), with Next/Previous buttons, step indicators (1/2/3 etc.), and validation per step.

Form steps (recommended 3 steps):
Step 1: Player Details
- Player full name (required)
- DOB (required, date)
- Gender (select: Male/Female/Other/Prefer not to say)
- Current school (required)
- Current school year (select: 3rd year / TY / 5th / 6th / Other)
- County (required)
- Current club (required)
- Playing position(s) (required; allow multiple or text)
- Level/league (optional)
Step 2: Football Profile + Links
- Highlight video link (required) + help text explaining how to share:
  “Upload to YouTube/Vimeo or Google Drive. If using Drive: Share -> Anyone with the link can view, or share directly with admissions email.”
- Optional extra links (max 3): match footage, CV, profile
- Notes about playing history/achievements (optional)
Step 3: Parent/Guardian Details + Consent
- Parent/guardian full name (required)
- Parent email (required)
- Parent mobile (required)
- How did you hear about us? (optional)
- Message (optional)
- Consent checkboxes (required):
  - “I am the parent/guardian and consent to submitting this application.”
  - “I understand we may contact me about the programme.”
- Anti-spam:
  - Honeypot hidden field
  - Simple rate limiting

On submit:
- POST to /api/apply
- Show loading state
- Redirect to /thank-you
